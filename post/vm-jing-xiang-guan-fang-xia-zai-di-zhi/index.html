安装后：在控制面板的添加删除Windows组件中关闭浏览器安全，避免每次访问都会提示安全问题。

网络连接问题：(常用1.NAT及2.桥接)
1.使用VMnet8可以组建局域网使用(不可做服务器不可联网)，主机和虚拟机可以相互ping通（VMnet8的ipv4）。
2.直连物理网络有两种方式。
aaa:第一种方式是将虚拟机的网络设置为桥接Bridge。这种模式下，虚拟机就可以获取实体网络中路由器分配的独立IP地址，从而连入实体网络。这时，虚拟机和物理机通过同一根网线接入路由器设备，共享路由器的同一个端口。
在我们以前的端口偷窃（Port Stealing）文章中提到，同一个端口只会和同一个IP地址进行绑定。如果虚拟机不发包，而实体机发包，路由器默认该端口绑定的是实体机。这时，如果有其他设备访问虚拟机的IP，就会出现无法响应的情况。这也是使用桥接方式的虚拟机，经常无法被其他实体机发现的原因。
bbb:为了网络稳定，可以采用第二种方式。第二种方式就是使用USB接口的有线网卡。让虚拟机直接使用USB有线网卡，而该有线网卡使用独立一根网线连接路由设备。这样避免了端口抢夺问题。

备注：
NAT方法：配置主机的VMint8->配置虚拟机的VMint8(ip网段与主机VMint8网段一致)->虚拟机服务器网络适配器选择NAT链接->进入虚拟机服务器配置ip（ip网段与主机VMint8网段一致）。

桥接方法：1.虚拟机->编辑->虚拟网络编辑器->更改配置（右下角）->VMnet0修改桥接模式并桥接至与主机联网描述一致的选项（如下图）->确定。


2.选择一个虚拟机服务器（开启的）右键设置->网络适配器->勾选前四个（已连接、启动时链接、桥接模式、复制物理网络链接状态）
3.进入虚拟机服务器后右键网上邻居打开属性->右键本地连接属性->双击Internet 协议（TCP/IP）->IP地址与主机ipv4同一网段即可，默认网关192.168.1.1，DNS与主机一样（备用填8.8.8.8（谷歌DNS服务器））->确定->打开cmd->ping www.baidu.com返回4个数据丢失0即完成联网配置。

（详细步骤可参考（桥接）：https://blog.csdn.net/vicoqi/article/details/79951236）

kail桥接联接wife使用命令配置kail的ipv4:
1.配置ip、子网掩码、默认网关：gedit /etc/network/interfaces

代码：
auto eth0
iface eth0 inet static
address 192.168.x.x
gateway 192.168.x.x
netmask 255.255.255.0

2.配置DNS：gedit /etc/resolv.conf

代码：
nameserver 8.8.8.8
nameserver 223.5.5.5
nameserver 180.76.76.76
备注：
8.8.8.8是谷歌的IP地址
223.5.5.5是阿里云的IP地址
180.76.76.87是百度的IP地址

3.启动eth0：ifconfig eth0 up

4.重启网络：service networking restart

5.检验成果：ifconfig 发现eth0的IP地址出现了。ping www.baidu.com ,成功返回。打开KALI浏览器，访问成功。
(备注：ctrl+c取消一些，例如中断ping)
